第一步 b
✅ 1. Cygwin 安装了 make 工具
请检查你是否安装了 make：
在 Cygwin 终端中运行：
make –version
如果返回 make 的版本号，说明你已经装好了。如果提示 “command not found”，你需要安装它。

✅ 安装 make（如果没装的话）：
运行 Cygwin 安装程序（通常是 setup-x86_64.exe）。
搜索并勾选安装以下包：

安装完后重新打开一个 Cygwin 终端。

✅ 2. 加载 Vivado 环境变量

在 Cygwin 中，你需要调用 Vivado 2018.3 的设置脚本，但注意是 Windows 路径形式，需要用 Cygwin 的路径转换格式：
2018.3

2021.1

这样 Vivado 的命令（比如 vivado, xvlog, xsim）才会在路径中。

✅ 3. 进入 HDL 工程目录并执行 make

2021.1          cd /cygdrive/d/Xilinx/hdl-hdl_2021_r1/projects/fmcomms2/zc706

⚠️ 注意路径前要加 /cygdrive/c/ 而不是 C:/，因为在 Cygwin 下这是 Linux 方式的路径表示。

如果出现rm.exe: lstat‘*.xxx’失败: Invalid argument

这是因为 Cygwin 和 Vivado SDK 中的 rm.exe 冲突了。

Vivado 自带了一个旧的 GNU 工具链（C:\Xilinx\SDK\2018.3\gnuwin\bin），这个路径可能在你的 PATH 中 优先级高于 Cygwin 自己的 rm，所以 Cygwin 脚本调用的是 Vivado 的 rm.exe，而它 不支持 POSIX 风格的通配符（如 *.cache）。

✅ 解决方法：屏蔽 Vivado SDK 自带的 rm.exe

在 Cygwin 中运行下面这行命令（临时生效）：

``

这会让 Cygwin 的 rm 和 make 等工具优先于 Vivado SDK 里的版本。

第二步
用vivado打开对应的程序，生成bit流打开sdk
创建一个新的空的App
No-OS 解压后，将\AD9361\no-OS-2018_R2\ad9361 文件夹下的sw文件全部复制到 SDK 中工程中的src 文件中 AD9361\hdl-hdl_2018_r2\projects\fmcomms2\AD9361\fmcomms2_zc706.sdk\AD9361\src
复制前

因为是 No-OS我们只需要xilinx和console的文件夹以及后面.c .h文件，其他操作系统和软件的文件夹不要

确保你移植的代码能在 SDK 2018.3 + Vivado 2018.3 + ZC706 + AD-FMCOMMS3-EBZ 的环境中正常编译运行。

右键AD9361 在C/C++Build setting中添加如下三个文件路径，避免头文件找不到

第三步 生成BOOT.BIN镜像烧写进SD卡

1.zc706_fbls.elf：”C:\Xilinx\hdl-hdl_2018_r2\projects\fmcomms2\zc706\fmcomms2_zc706.sdk\zc706_fbls\Debug\zc706_fbls.elf”
2.system_top.bit：”C:\Xilinx\hdl-hdl_2018_r2\projects\fmcomms2\zc706\fmcomms2_zc706.runs\impl_1\system_top.bit”
3.AD9361.elf：”C:\Xilinx\hdl-hdl_2018_r2\projects\fmcomms2\zc706\fmcomms2_zc706.sdk\AD9361\Debug\AD9361.elf”
建议把他们几个放在同一个目录下，比如创建一个专门用于输出镜像文件的文件夹boot.img,放在zc706根目录下

这三个顺序不要错

FPGA加载依赖​system_top.bit作为Xilinx FPGA配置文件，需先于ARM处理器（zc706_fbls.elf/AD9361.elf）加载。若顺序颠倒可能导致
硬件逻辑未初始化即运行软件、DDR内存控制器未配置导致数据异常、外设接口（如AD9361）寄存器地址映射失败
​启动链式依赖​
典型Zynq架构启动流程要求：BootROM → FSBL(如zc706_fbls.elf) → Bitstream(system_top.bit) → u-boot/App(AD9361.elf)
顺序错乱会导致：加载器无法找到有效镜像、片上系统(OSPI/QSPI)读取中断、内存地址冲突
​交叉编译关联性​，各文件生成依赖关系决定顺序：system_top.bit需先于应用ELF编译、AD9361.elf可能依赖FPGA生成的DMA/IP核、路径不一致会导致链接器找不到符号定义）

然后点击create image生成就可以了

现在可以进行烧写进SD卡了

最后就可以点击program烧写了
